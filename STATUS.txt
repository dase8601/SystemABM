THE THESIS
----------
"Real intelligence starts in the world, not language."
  — Dupoux / LeCun / Malik 2026

The claim: a frozen JEPA-class encoder (pretrained on passive observation,
no pixel reconstruction) + action-conditioned world model predictor +
autonomous mode switching will outperform raw RL on 3D visual navigation.

Paper 1 proved the switching mechanism works (DoorKey: 70% > 62% > 24%).
Paper 2 proves pretrained abstract representations give a measurable advantage.

Yann LeCun direct quotes (Adam Brown interview, 2026) that validate our work:

  "The system learns an abstract representation of what it observes and it
  makes prediction in that abstract representation space. This is really
  the way humans and animals function."

  "It will not be LLM. It will not be generative models that predict discrete
  tokens. It will be models that learn abstract representations and make
  predictions in abstract representations and can reason about what is going
  to be the effect of me taking this action."

  "DINOv2 is probably the best image encoder that we have at the moment."
  — THIS IS OUR SYSTEM A ENCODER. Yann's own words confirm thesis alignment.


WHAT WE BUILT
-------------

System A — Frozen JEPA Encoder + World Model Predictor
  Encoder: DINOv2 ViT-B/14
    - 86M params, 768-dim CLS token, L2-normalized
    - Input: 160x160 RGB -> resized to 224x224 -> ImageNet normalized
    - Output: 768-dim float32 via forward_features(x)["x_norm_clstoken"]
    - Loaded: torch.hub.load("facebookresearch/dinov2", "dinov2_vitb14", pretrained=True)
    - Feature quality confirmed: cos_sim(black,white) = 0.6195 (discriminative)
    - Code: abm/vjepa_encoder.py

  Predictor (VJEPAPredictor): action-conditioned MLP (768+3 -> 512 -> 512 -> 768)
    - Trained during OBSERVE phase: given (z_t, a_t), predict z_{t+1}
    - Prediction error = intrinsic reward during ACT phase
    - Code: abm/lewm.py

System B — PPO Reinforcement Learning
  - Standard PPO on 768-dim DINOv2 features
  - RND (Random Network Distillation) exploration bonus in standalone conditions

System M — Autonomous Meta-Controller
  - FSM: OBSERVE mode <-> ACT mode
  - Switch OBSERVE->ACT: when ssl_ewa drops (predictor converged)
  - Switch ACT->OBSERVE: when extrinsic reward plateau detected
  - Code: abm/loop.py

Environment — MiniWorld-MazeS3
  - 3D procedurally generated maze, first-person ego-centric RGB 160x160
  - 3 discrete actions: turn left, turn right, move forward
  - 16 parallel envs via SyncVectorEnv (NOT AsyncVectorEnv — see below)
  - Code: abm/miniworld_env.py


WHAT WE TRIED AND WHY WE CHANGED
---------------------------------

V-JEPA 2.1 ViT-B (original encoder — FAILED)
  - Loaded correctly: 158/158 keys matched, no errors
  - Problem: cos_sim(black,white) = 0.9969 — complete feature collapse
  - Root cause: mean pooling 576 patches creates massive anisotropy in 768-dim
    space. Every image maps to roughly the same direction regardless of content.
  - Result: 350K training steps at 0-5% success, predictor loss flat at 0.007
  - Attempted fix: max pool + L2 normalize -> cos_sim still 0.9969. No help.
  - Conclusion: V-JEPA single-frame image mode architecturally unsuitable for RL.

DINOv3 ViT-B/16 (attempted upgrade — BLOCKED)
  - Tried HuggingFace transformers -> 401 Unauthorized (gated, needs Meta approval)
  - Tried torch.hub -> GitHub code downloads fine, but weights at
    dl.fbaipublicfiles.com return 403 Forbidden. Fully gated.
  - Reverted to DINOv2 permanently.

DINOv2 ViT-B/14 (final encoder — WORKING)
  - cos_sim(black,white) = 0.6195 — strongly discriminative
  - IS a JEPA-class method: self-supervised, no pixel reconstruction, 1.2B images
    of passive observation, joint embedding (self-distillation + local/global crop)
  - Zero changes to VJEPAPredictor (same 768-dim feature_dim)

Habitat-sim (environment candidate — DEFERRED to Paper 3)
  - Requires 10-30 min C++ compilation on RunPod + habitat-lab (not downloaded)
  - MiniWorld handles Papers 1-2. Habitat is Paper 3.

KEY INSIGHT FOR THE PAPER
--------------------------

V-JEPA failed and DINOv2 succeeded — not because V-JEPA is bad, but because of
a mismatch between architecture and use case. V-JEPA was designed to model temporal
dynamics across video frames. In single-frame image mode, its patch features all
activate similarly regardless of content (anisotropy). DINOv2 was explicitly
trained on single-image discrimination via local/global crop self-distillation.

This IS the JEPA thesis in action: the right abstract representation for the task
matters. DINOv2, trained on 1.2B images of passive observation, produces the
abstract spatial representations that support downstream navigation reasoning.

Yann: "We find abstractions that allow us to make predictions while ignoring all
the details we cannot predict." DINOv2's CLS token is that abstraction.
